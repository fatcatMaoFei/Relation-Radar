# Relation Radar

Relation Radar 是一个本地优先的 AI 关系助手，把每个朋友当作可以持续更新的「活数据库」，帮助你在繁忙生活中记住人、读懂情绪、避免踩雷。

它更像是一个面向人际关系的「第二大脑」——你记录事实和细节，AI 基于这些记录做检索、总结和提醒，帮你在重要时刻说对话、做对事。

## 核心理念

- **只记录事实，推理交给 AI**  
  你记录发生了什么、谁在场、当时说了什么、喜欢/讨厌什么；  
  AI 负责做情绪判断、关系趋势分析和场景化建议（如吃饭选店、送礼建议）。

- **本地优先，隐私可控**  
  所有原始记录和向量索引都保存在本地 SQLite / 向量库中，默认使用本地小模型；  
  远端大模型（如果启用）只能通过受控接口访问「脱敏后的摘要」，而不是你完整的聊天记录。

- **人际关系，而不是监控**  
  Relation Radar 不做爬虫、不自动抓社交平台数据，只处理你主动输入的文本 / 截图 / 语音转写。  
  它帮助你更体贴、更有记忆，而不是去「审判」或「控制」任何人。

## 能帮你做什么？

- **记住细节**  
  - 谁喜欢什么菜系、什么礼物、什么聊天风格；  
  - 谁怕冷怕热、对哪些话题敏感；  
  - 最近谁说过「压力大」「想被需要」「很开心」。  

- **在合适的时候做合适的事**  
  - 聚餐前问：「这次和猫、阿 B 一起吃饭，有什么推荐？」 → 给出菜系/环境建议和注意事项；  
  - 生日前问：「猫生日送什么不会踩雷？」 → 基于历史偏好和忌讳给出 1–3 个礼物思路；  
  - 短期总结：「最近谁情绪不太好，我是不是该关心一下？」。

- **回看关系的轨迹**  
  - 按时间线回顾你和某人的所有重要事件和情绪变化；  
  - 分析长期趋势，比如「这一年你们吃饭/聊天/吵架的频率有何变化」；  
  - 通过多条记录，得到比单次印象更客观的关系感受。

## 产品特点

- **统一的事件模型**  
  不管信息来自手动输入、聊天截图还是语音转写，最终都会沉淀为带有「人物、时间、情绪、偏好、忌讳、标签」等字段的事件（Event）。

- **RAG + 本地大模型**  
  查询时会先用向量检索（RAG）找到与你问题最相关的一批事件，再交给本地或本地+远端大模型做自然语言回答，尽量「有据可依，不乱编」。

- **可调的「记忆范围」**  
  平时问一个具体问题，只需查最近 5～10 条相关记录就够；  
  想看趋势或做总结时，可以在界面里主动调大检索范围（比如 30 条），让 AI 参考更长的历史。

- **脚本友好，界面逐步完善**  
  目前已有命令行界面（CLI），支持添加人物、录入事件、提问和调试；  
  Web 界面会逐步实现「一人一本笔记本」的布局：左侧是人物列表，中间是按时间/标签筛选的事件流，右侧是问答区域。

## 使用边界与期待

- Relation Radar 不保证「客观真实」，只能基于你自己记录的片段做推理和建议；  
- 不建议用于做决策的唯一依据，而是用来补充记忆、提供参考；  
- 希望它是一个帮你更真诚、更细腻对待身边人的小工具，而不是让关系变得功利的打分系统。
